Usar instrucciones con el controlador JDBC

El controlador JDBC de Microsoft SQL Server se puede usar para trabajar con los datos de una base de datos de SQL Server de distintos modos. El controlador JDBC se puede usar para ejecutar instrucciones de SQL contra la base de datos o para llamar a procedimientos almacenados en la base de datos, empleando parámetros de entrada y de salida. El controlador JDBC también admite el uso de secuencias de escape de SQL, claves generadas automáticamente y actualizaciones realizadas dentro de una operación por lotes.

El controlador JDBC proporciona tres clases para la recuperación de datos desde una base de datos de SQL Server.

  1. SQLServerStatement, para ejecutar instrucciones de SQL sin parámetros.
  2. SQLServerPreparedStatement - (heredada de SQLServerStatement), para ejecutar instrucciones de SQL compiladas que pueden contener parámetros de entrada.
  3. SQLServerCallableStatement, (heredada de SQLServerPreparedStatement), para ejecutar procedimientos almacenados que pueden contener parámetros IN, OUT o de ambos tipos.

Los temas de esta sección tratan cómo usar cada una de estas tres clases de instrucciones para trabajar con datos de una base de datos SQL Server.
