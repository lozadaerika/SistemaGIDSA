Requisitos del sistema para el controlador JDBC

Para tener acceso a los datos desde una base de datos de SQL Server mediante el controlador JDBC de Microsoft SQL Server, debe tener los siguientes componentes instalados en el equipo:

  • Controlador de JDBC de Microsoft SQL Server
  • Java Runtime Environment

Requisitos de Java Runtime Environment

Desde el controlador JDBC de Microsoft SQL Server, el controlador JDBC proporciona compatibilidad con la API de Java Database Connectivity (JDBC) Spec que se ha ampliado para incluir la API de JDBC 4.0. El controlador JDBC 4.0 se incluyó como parte de Sun Java SE Development Kit (JDK) 6.0 y Java Runtime Environment (JRE) 6.0. JDBC 4.0 es un superconjunto de JDBC 3.0.

El controlador JDBC de Microsoft SQL Server es compatible con JDBC 3.0 y JDBC 4.0.

Cuando implementa el controlador JDBC de Microsoft SQL Server en los sistemas operativos Windows y UNIX, debe usar respectivamente los paquetes de instalación sqljdbc_<versión>_enu.exe y sqljdbc_<versión>_enu.tar.gz. Para obtener más información acerca de cómo implementar el controlador JDBC, vea el tema Implementar el controlador JDBC.

Para la compatibilidad con versiones anteriores y los posibles escenarios de actualización, el controlador JDBC incluye las dos bibliotecas de clases JAR de cada paquete de instalación: sqljdbc.jar y sqljdbc4.jar.

JAR Descripción

sqljdbc.jar

La biblioteca de clases sqljdbc.jar proporciona compatibilidad con JDBC 3.0.

La biblioteca de clases sqljdbc.jar necesita la versión 5.0 de Java Runtime Environment (JRE). Usar sqljdbc.jar en JRE 6.0 generará una excepción al conectar con una base de datos.

Nota: El controlador JDBC no es compatible con JRE 1.4. Debe actualizar JRE 1.4 a JRE 5.0 o una versión posterior al usar el controlador JDBC. En algunos casos puede necesitar volver a compilar la aplicación, por no ser compatible con JDK 5.0 o una versión posterior. Para obtener más información, visite el sitio web de Sun Microsystems.

sqljdbc4.jar

La biblioteca de clases sqljdbc4.jar proporciona compatibilidad con JDBC 4.0. Incluye todas las características de sqljdbc.jar y los métodos nuevos de JDBC 4.0.

La biblioteca de clases sqljdbc4.jar necesita la versión 6.0 de Java Runtime Environment (JRE) o una posterior. Usar sqljdbc4.jar en JRE 1.4 o 5.0 generará una excepción.

Nota: Use sqljdbc4.jar cuando la aplicación deba ejecutarse en JRE 6.0, incluso aunque no use las características de JDBC 4.0.

Tenga en cuenta que el controlador JDBC se ha diseñado para la compatibilidad y colaboración con todas las máquinas virtuales Java importantes de Sun, pero se comprobó en Sun JRE 5.0 o posterior.

Requisitos de SQL Server

El controlador JDBC se ha diseñado especialmente para usar las características incluidas en SQL Server 2005, aunque es compatible con versiones anteriores a SQL Server 2000, incluyendo la versión de 64 bits. El controlador JDBC 3.0 admite los tipos nuevos de fecha y hora, los tipos definidos por el usuario de gran tamaño y las columnas dispersas en SQL Server 2008.

Requisitos del sistema operativo

El controlador JDBC se ha diseñado para funcionar en cualquier sistema operativo que admita el uso de una máquina virtual Java (JVM). No obstante, solo se han probado oficialmente los sistemas operativos Sun Solaris, SUSE Linux y Windows XP o sistemas operativos posteriores.

Lenguajes compatibles

El controlador JDBC es compatible con todas las intercalaciones de columnas de SQL Server. Para obtener más información acerca de las intercalaciones compatibles con el controlador JDBC, consulte Características internacionales del controlador JDBC.

Para obtener más información acerca de las intercalaciones, consulte "Trabajar con intercalaciones" en los Libros en pantalla de SQL Server.
